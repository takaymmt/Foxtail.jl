# Foxtail.jl — Indicator Architecture Analysis

> Generated by Architect subagent (Opus). Date: 2026-04-01

## Key Findings

1. **Four macro patterns govern all indicators**: `@prep_siso` (Vector→Vector), `@prep_miso` (Matrix→Vector), `@prep_simo` (Vector→Matrix), `@prep_mimo` (Matrix→Matrix). Every new indicator must end with exactly one of these macro calls. The macros auto-generate TSFrame wrappers and exports.

2. **Sub-indicator composition is the dominant pattern**: MACD calls EMA 3x, ChaikinOsc calls ADL+EMA, StochRSI calls RSI+MinMaxQueue+apply_ma, BB calls SMA_stats/EMA_stats. New indicators should maximally reuse existing building blocks (EMA, SMA, ATR/TR, apply_ma, ADL, MinMaxQueue, CircBuff).

3. **`apply_ma(data, ma_type; n)` is the universal smoothing dispatcher**: supports :SMA, :EMA, :SMMA/:RMA, :WMA. Any new indicator needing configurable smoothing should use this rather than calling individual MAs directly.

4. **MinMaxQueue enables O(1) sliding min/max**: already used by Stoch, WR, StochRSI. Donchian Channel, Aroon, and Ichimoku can directly reuse it. For Aroon, a small `get_max_idx()`/`get_min_idx()` accessor may be needed.

5. **TR() is available module-wide without export**: defined in ATR.jl, callable from any indicator file. DMI/ADX, Vortex, and Ultimate Oscillator can call it directly.

6. **10 of 23 TODO indicators are quick wins (< 50 LOC, high reuse)**: Keltner Channel (EMA+ATR), Donchian Channel (MinMaxQueue), Supertrend (ATR), ROC (arithmetic), PPO (EMA), CCI (CircBuff+SMA), Aroon (MinMaxQueue), VPT/NVI/PVI (accumulators).

7. **Test pattern is consistent and strict**: every indicator needs type checks, TSFrame checks, column name checks, parameter variation tests, numerical validation with hardcoded expected values (atol tolerance), and property/boundary tests.

## Complexity Categorization

### Tier 1 — Trivial (< 30 LOC)
- **ROC**: SISO, pure arithmetic `(P_t - P_{t-n}) / P_{t-n} * 100`
- **DPO**: SISO, `price - shifted SMA`
- **NVI/PVI**: MISO [Close,Volume], cumulative accumulators (OBV-like)
- **VPT**: MISO [Close,Volume], `VPT += V * (deltaP/P)`

### Tier 2 — Easy (30-60 LOC, reuses existing)
- **Keltner Channel**: MIMO, `EMA ± mult × ATR` — both exist
- **Donchian Channel**: MIMO, MinMaxQueue for rolling max/min
- **Supertrend**: MIMO, ATR-based bands + simple trend flip state
- **PPO**: SIMO, percentage MACD using EMA
- **CCI**: MISO, typical price + SMA + mean absolute deviation
- **Aroon**: MIMO, bars since max/min using MinMaxQueue variant
- **Force Index**: MISO, `EMA(deltaClose × Volume)`
- **EMV**: MISO, distance moved / box ratio + SMA smoothing

### Tier 3 — Medium (60-100 LOC)
- **DMI/ADX**: MIMO, directional movement + TR + SMMA smoothing
- **MFI**: MISO, RSI-like on money flow (pos/neg accumulators)
- **CMF**: MISO, rolling sum of money flow volume / volume
- **VWAP**: MISO, cumulative TP×V / cumulative V
- **Vortex**: MIMO, directional sums / TR sums over window
- **Mass Index**: MISO, sum of EMA(range)/EMA(EMA(range))
- **KST**: SIMO, weighted sum of 4 smoothed ROCs (needs ROC first)

### Tier 4 — Complex (100+ LOC)
- **Ichimoku Cloud**: MIMO, 5 output lines with time shifts
- **Parabolic SAR**: MIMO, state machine with acceleration factor
- **Ultimate Oscillator**: MISO, 3 timeframe buying pressure / TR ratios
- **VIX/Historical Volatility**: SISO, rolling std dev of log returns
